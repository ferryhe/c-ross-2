# 专业回答引擎演进记录

## 目标

在不破坏现有独立 chatbot 的前提下，把项目逐步升级为：

`面向监管规则知识库的、可验证、可扩展、适合 AI agent 调用的专业回答引擎`

## 本轮演进内容

### 1. 保留独立 chatbot

- 保留 `/api/chat`
- 不改变当前前端默认工作模式
- 不把专业能力硬塞进当前聊天接口

### 2. 新增专业引擎骨架

- 新增 `regulatory_engine.py`
- 增加 title search
- 增加 summary search
- 增加 query planning
- 增加专业引擎聊天入口

### 3. 新增 skill 资产

- 新增 `skills/regulatory-markdown-answering/SKILL.md`
- 明确 agent 处理法规 Markdown 的固定流程

### 4. 新增多份设计文档

- `详细对比说明.md`
- `专业回答引擎改造计划.md`
- `AI_Agent/专业回答引擎API说明.md`
- 本文档

### 5. 新增 ready-data 与真实评测闭环

- 新增 `AI_Agent/scripts/build_ready_data.py`
- 产出 `Knowledge_Base_MarkDown/ready_data/` 第一版数据层产物
- 新增 `AI_Agent/scripts/evaluate_regulatory_answers.py`
- 用 `gpt-5.4-mini` 对专业引擎和当前 agentic chatbot 做真实问答验证
- 新增 `AI_Agent/真实问答验证记录-gpt-5.4-mini.md`

## 当前架构认识

当前已经具备：

- 规则 / 通知 / 附件 Markdown 知识库
- hybrid document/section retrieval
- 公式渲染与清洗
- 引用式回答
- ready-data 主产物
- 真实问答回归脚手架

当前还需要增强：

- 通知覆盖关系细化
- 公式变量说明增强
- 规则问题中的通知噪声收缩
- 更细粒度的 agent-ready tool contract

## 预期下一步

### 近一步

- 在索引构建阶段打更多结构化标签
- 为规则与通知生成稳定 summary sidecar
- 为公式与阈值建立独立检索能力

### 再下一步

- 补前端“专业引擎模式”开关
- 补 agent tools 级接口
- 把通知覆盖关系纳入回答链路

## 提交策略

本分支尽量按以下思路拆分提交：

1. 引擎代码和 API 骨架
2. 测试与兼容性
3. 技能与规划文档
4. API 说明和演进记录

这样更方便审阅，也更方便之后做 cherry-pick 或回退。
