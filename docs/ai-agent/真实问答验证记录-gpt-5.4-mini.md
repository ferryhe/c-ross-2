## 评测目的

验证当前仓库是否已经具备：

- 从法规 Markdown 产出可消费的数据层
- 基于该数据层和现有 RAG 层回答关键监管问题
- 在保留当前 chatbot 的情况下，支持更稳定的专业引擎回答

## 本轮关键结论

1. 专业引擎的 6 个关键问题均能给出可接受且基本正确的答案。
2. 当前独立 agentic chatbot 这 6 个问题也都能回答，但专业引擎在“问题 scope 收缩”上更稳定。
3. 新增的“编号精确命中 + 标题主题命中”作用域策略，明显改善了：
   - `最低资本由哪些部分组成？`
   - `控制风险最低资本按照哪一项规则计量？`
4. 当前剩余的主要风险不是“答不出来”，而是某些规则问题仍会混入通知类辅助证据；答案通常仍然正确，但证据集合还可以继续收紧。

## 人工判定结果

### 1. `rule_count`

问题：

`偿二代二期监管规则一共有多少号规定？`

结论：

- 专业引擎：正确
- agentic chatbot：正确

判断：

- 两者都直接从 manifest / catalog 层回答
- 给出结论为：当前知识库共收录 `20` 项，编号从第 `1` 号到第 `20` 号

### 2. `rule2_overview`

问题：

`保险公司偿付能力监管规则第2号主要涉及什么内容？`

结论：

- 专业引擎：正确
- agentic chatbot：正确

判断：

- 两者都正确识别出第2号规则是“最低资本”
- 回答覆盖了：
  - 适用范围
  - 风险分类
  - 最低资本三部分构成
  - 计量原则
  - 计量方法和公式
- 专业引擎已经能把 scope 锁到 `规则第2号`

### 3. `minimum_capital_components`

问题：

`最低资本由哪些部分组成？`

结论：

- 专业引擎：正确
- agentic chatbot：正确

- 调整后已经优先落到 `规则第2号：最低资本`
- 两者都给出了正确三分法：
  - 可资本化风险最低资本
  - 控制风险最低资本
  - 附加资本

### 4. `rule2_formula`

`规则第2号里最低资本的计算公式是什么？请解释变量含义。`

### 5. `attachment4_summary`

`附件4是什么，主要包含什么内容？`

- agentic chatbot：正确

  - 财产保险公司部分

`控制风险最低资本按照哪一项规则计量？`

## 当前最有价值的认识

### 1. 当前 repo 已经不仅能“做 chatbot”

它已经开始具备：

- 标题定位
- 摘要缩圈
- section / formula 级证据回答
- 附件与规则关系回答

这说明 repo 正在从“可问答 demo”进入“可复用数据层 + 可验证回答引擎”阶段。

### 2. ready-data 对专业引擎已经开始起作用

本轮最明显的收益点有两个：

- 精确编号问题不再轻易跑偏
- 非精确但主题明确的问题，也能通过标题主题词锁定更合理的主文档

### 3. 还可以继续优化证据纯度

虽然答案已经大多正确，但对规则类问题：

- 通知类片段仍会进入部分答案上下文
- 关系图目前还是轻量版
- 对“通知覆盖哪一条规则、改了哪个公式”还可以继续结构化

## 下一步建议

1. 继续给 `relations_graph.json` 增加更强语义边：
   - `clarifies_formula`
   - `adjusts_rule`
   - `extends_transition`
2. 在 `formula_cards.jsonl` 中补变量别名和适用范围
3. 对“规则号已明确”的问题进一步减少通知类辅助证据噪声
4. 把这组关键问题固化成回归测试基线
