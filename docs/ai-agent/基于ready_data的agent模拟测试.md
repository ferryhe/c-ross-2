# 基于 ready_data 的 agent 模拟测试

日期：2026-04-03

## 测试目的

模拟一个不直接读取原始 Markdown、而是优先使用现有 `ready_data` 的 AI agent。

本次只使用以下产物：

- `Knowledge_Base_MarkDown/ready_data/doc_catalog.jsonl`
- `Knowledge_Base_MarkDown/ready_data/doc_summaries.jsonl`
- `Knowledge_Base_MarkDown/ready_data/sections_structured.jsonl`
- `Knowledge_Base_MarkDown/ready_data/formula_cards.jsonl`

不额外调用在线模型，不额外跑全文检索。

## 结论先看

这套 `ready_data` 已经足够支撑几类典型问题：

- 目录/数量类
- 单规则概览类
- 条文明确结论类
- 附件概览类
- 公式定位类
- 规则关系类

其中：

- `doc_summaries` 很适合做“这份规则主要讲什么”
- `sections_structured` 很适合做“哪一条明确写了什么”
- `formula_cards` 适合做“公式定位”
- 真正要解释公式变量含义时，仍然最好联动 `sections_structured`

## 模拟问答

### 1. 问题：偿二代二期监管规则一共有多少号规定？

使用的数据：

- `doc_catalog.jsonl`

模拟答案：

根据当前 `ready_data` 里的文档目录，`rules` 类文档共 20 份，因此当前知识库收录的偿二代二期监管规则编号范围是第 1 号到第 20 号。

评价：

- 效果好
- 这类问题只需要目录层，不需要全文层

### 2. 问题：保险公司偿付能力监管规则第2号主要涉及什么内容？

使用的数据：

- `doc_summaries.jsonl`
- 目标文档：`rules/保险公司偿付能力监管规则第2号：最低资本.md`

模拟答案：

规则第 2 号的主题是“最低资本”。从摘要和 focus points 看，这份规则主要讲四块内容：

- 最低资本的定义和构成
- 最低资本的计量原则
- 最低资本的计量方法
- 附则

更具体地说，它首先说明最低资本是为了应对各类可资本化风险对偿付能力的不利影响而要求保险公司具备的资本数额；然后规定最低资本由可资本化风险最低资本、控制风险最低资本和附加资本组成；接着说明计量要以风险为基础、采用相关系数矩阵法，并在第三章给出具体计量方法。

评价：

- 效果好
- 对“概览类问题”，`doc_summaries` 已经足够形成一个比较自然的回答

### 3. 问题：最低资本由哪些部分组成？

使用的数据：

- `sections_structured.jsonl`
- 目标 section：`rules/保险公司偿付能力监管规则第2号：最低资本.md#section-1`

模拟答案：

最低资本由三部分组成：

- 可资本化风险最低资本
- 控制风险最低资本
- 附加资本

其中：

- 可资本化风险最低资本对应保险风险、市场风险、信用风险
- 控制风险最低资本对应控制风险
- 附加资本包括逆周期附加资本、系统重要性保险机构附加资本以及其他附加资本

评价：

- 效果很好
- 这类“条文已经直接给出结论”的问题，`sections_structured` 比摘要层更可靠

### 4. 问题：规则第2号里最低资本的计算公式是什么？变量是什么意思？

使用的数据：

- `formula_cards.jsonl`
- 目标公式：`rules/保险公司偿付能力监管规则第2号：最低资本.md#formula-1`
- `sections_structured.jsonl`
- 目标 section：`rules/保险公司偿付能力监管规则第2号：最低资本.md#section-3`

模拟答案：

规则第 2 号给出的可资本化风险整体最低资本公式是：

```math
\mathrm{MC}^{*} = \sqrt{\mathrm{MC}_{\text{向量}} \times \mathrm{M}_{\text{相关系数}} \times \mathrm{MC}_{\text{向量}}^{\mathrm{T}}} - \mathrm{LA}
```

变量含义可以直接从结构化 section 中得到：

- `MC^*`：可资本化风险整体的最低资本
- `MC_向量`：保险风险、市场风险和信用风险的最低资本行向量
- `M_相关系数`：相关系数矩阵
- `LA`：特定类别保险合同的损失吸收效应调整

评价：

- 效果好
- 也说明一个很重要的点：`formula_cards` 负责“找到公式”，`sections_structured` 负责“解释公式”
- 如果 agent 只看 `formula_cards`，变量解释还不够稳；联动 section 后就够用了

### 5. 问题：附件4是什么，主要包含什么内容？

使用的数据：

- `doc_summaries.jsonl`
- 目标文档：`attachments/附件4：保险公司压力测试必测压力情景和必测因素.md`

模拟答案：

附件 4 是《保险公司压力测试必测压力情景和必测因素》。

从摘要和 headings 看，它主要讲的是保险公司压力测试中必须覆盖的压力情景和敏感性测试因素，核心结构包括：

- 财产保险公司必测压力情景和必测因素
- 年度压力测试必测压力情景
- 保险风险情景
- 宏观经济风险情景
- 年度敏感性测试必测因素

也就是说，附件 4 不是讲一般原则，而是更偏“压力测试必须怎么测、至少要测哪些情景和因素”。

评价：

- 效果好
- 附件类问题用摘要层就能形成比较像样的回答

### 6. 问题：控制风险最低资本按照哪一项规则计量？

使用的数据：

- `sections_structured.jsonl`
- 目标 section：`rules/保险公司偿付能力监管规则第2号：最低资本.md#section-4`
- `doc_summaries.jsonl`
- 目标文档：`rules/保险公司偿付能力监管规则第12号：偿付能力风险管理要求与评估.md`

模拟答案：

控制风险最低资本按照《保险公司偿付能力监管规则第12号：偿付能力风险管理要求与评估》计量。

从第 2 号规则第三章的结构化 section 可以直接看到第二十条写明了这一点；而第 12 号规则的摘要也进一步说明，它本身就是用来规范风险管理能力评估并确定控制风险最低资本计量标准的。

评价：

- 效果很好
- 这是一个很典型的“关系类问题”，但当前轻量数据层已经够用

## 这次测试说明了什么

### 1. 现在的数据层已经能支撑一个“轻量 agent”

至少在下面这些问题上，agent 不必每次都回原始 Markdown 全文：

- 先定位规则
- 先给概览
- 先回答明显条文结论
- 先定位附件
- 先定位公式

### 2. 不是所有问题都需要更多加工

这次结果说明：

- `doc_catalog + doc_summaries + sections_structured + formula_cards`
- 已经是一个可用的最小组合

所以当前不需要再大规模加字段。

### 3. 真正还需要增强的地方也很清楚

如果后面发现效果不好，最值得增量改的是：

- 通知与规则之间的覆盖/调整关系
- 公式变量别名和适用范围
- 某些通知文档摘要里图片噪声偏多的问题

## 总体判断

如果把当前 repo 的目标定义为：

`把监管 Markdown 加工成 chatbot-ready / AI-agent-ready 的数据层`

那现在这套 `ready_data` 已经达到了“可以先拿去给 agent 用”的程度。

下一步更合适的策略不是继续重加工，而是：

1. 先让 agent 真用这套数据层回答
2. 观察哪些问题答不好
3. 再针对性补结构
