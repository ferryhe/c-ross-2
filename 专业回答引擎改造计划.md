# 专业回答引擎改造计划

目标：在保留当前可独立运行 chatbot 的前提下，把项目逐步演进为 `面向监管规则知识库的、可验证、可扩展、适合 AI agent 调用的专业回答引擎`。

## 当前已落地的第一步

- 保留现有 `/api/chat` 作为独立 chatbot 入口。
- 新增法规专业引擎骨架：
  - `AI_Agent/scripts/regulatory_engine.py`
  - `/api/engine/config`
  - `/api/engine/search/titles`
  - `/api/engine/search/summaries`
  - `/api/engine/plan`
  - `/api/engine/chat`
- 新增 repo 内 skill：
  - `skills/regulatory-markdown-answering/SKILL.md`

## 分阶段计划

### 阶段 1：标题与摘要层

目标：

- 标题、编号、别名优先定位
- 文档摘要检索单独成层

已实现：

- manifest 驱动的 catalog 加载
- 标题搜索
- 摘要搜索
- 问题类型规划

后续补强：

- 增加更丰富的别名规则
- 增加人工维护的 `title_aliases`
- 增加更高质量的 structured summary 生成

### 阶段 2：结构化证据层

目标：

- 明确 article / formula / table / threshold / obligation / exception

后续任务：

- 在索引构建阶段补结构化 section 标签
- 给公式建立变量说明卡片
- 给表格、曲线、名单类附件加单独类型

### 阶段 3：关系图层

目标：

- 让引擎知道规则、通知、附件之间的关联和覆盖关系

后续任务：

- 建 `rule -> notice`
- 建 `rule -> attachment`
- 建 `notice -> adjusted_rule`
- 建 `formula -> variable_definition`

### 阶段 4：agent tools 化

目标：

- 让 AI agent 按步骤调用，而不是只打一条黑盒问答

建议工具：

- `search_titles`
- `search_summaries`
- `search_articles`
- `search_formulas`
- `search_tables`
- `expand_relations`
- `compose_regulatory_answer`

### 阶段 5：双模式产品化

目标：

- 保留独立 chatbot
- 同时提供专业引擎模式

模式划分：

- `chatbot mode`
  - 继续走当前 `/api/chat`
  - 面向通用问答和当前前端
- `professional engine mode`
  - 走 `/api/engine/*`
  - 面向 OpenClaw、skills、工作流 agent、专业前端

## 兼容性原则

1. 不破坏现有前端和 `/api/chat`。
2. 新能力优先以新增模块和新增 API 暴露。
3. 新引擎先做“可调用、可规划、可扩展”，再逐步加深结构化能力。
4. skill 与 API 要一一对应，避免只有 prompt 没有能力层。

## 下一轮最值得做的开发项

1. 在 `build_index.py` 中新增结构化 section 标签输出。
2. 为规则和通知生成更稳定的摘要 sidecar 文件。
3. 为公式和阈值建立独立检索层。
4. 在前端增加一个“专业引擎模式”切换，不影响当前默认 chatbot。
